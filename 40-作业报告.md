## 一、项目概述

MyMoneyBook 是一款基于 Qt 6 框架开发的跨平台个人财务管理桌面应用程序。项目采用 C++17 标准，以 SQLite 为本地数据存储引擎，利用 Qt Charts 模块实现数据可视化，旨在为个人用户提供简洁高效、功能完整的日常记账解决方案。

### 1.1 开发环境

| 项目 | 说明 |
|------|------|
| 编程语言 | C++17 |
| 框架 | Qt 6.11 |
| 构建系统 | CMake 3.16+ |
| 编译器 | MinGW 64-bit |
| 数据库 | SQLite 3（本地文件存储） |
| 图表库 | Qt Charts |
| 目标平台 | Windows / macOS / Linux |

## 二、程序功能

本系统共实现 **八大核心功能模块**，覆盖个人记账的完整业务流程。

### 2.1 用户注册与登录

- 支持用户名/密码注册与登录，通过 `QStackedWidget` 实现登录页与注册页的单窗口切换
- 密码采用 SHA-256 加盐哈希存储（盐值 `"MyMoneyBook_Salt_"`），不保存明文
- 输入验证：用户名至少 3 个字符，密码至少 4 个字符，注册需两次密码一致
- 注册成功后自动创建 1 个默认账户、15 个默认收支分类及其对应图标

### 2.2 多账户管理

- 支持现金、支付宝、银行卡、微信、其他五种账户类型
- 账户信息包含名称、类型、余额三个属性
- 支持账户的添加、修改、删除，余额以颜色区分正负
- 账单录入时可选择不同账户，系统自动维护各账户余额（收入增加余额，支出减少余额）
- 修改或删除账单时，自动回溯并更正对应账户的余额变动

### 2.3 账单管理

- 支持收支记录的增加、删除、修改，按时间倒序展示
- 每条记录包含：日期、类型（收入/支出）、账户、分类、金额、备注
- 金额列按收支类型着色（收入绿色 #00C897，支出红色 #FF6B6B）
- 顶部实时汇总显示总收入、总支出、结余，结余以蓝色（#5B7FFF）高亮
- 支持将全部账单导出为 CSV 文件（含账户列），方便在 Excel 中进一步分析

### 2.4 分类管理（含图标）

- 支持自定义收支分类的添加、修改、删除
- 注册时自动预置 5 个收入分类和 10 个支出分类，各有对应默认图标（如下表所示）
- 内置 100 个 emoji 图标选择器，用户可自由为每个分类挑选图标
- 分类图标在账单录入下拉框、搜索筛选下拉框中同步显示
- 收入/支出分类在表格中以不同颜色区分

| | | 教育 | 📚 |
| | | 通讯 | 📱 |
| | | 日用 | 🧴 |
| | | 其他支出 | 📦 |

### 2.5 预算管理

- 按月份设置月度支出预算金额
- 实时统计当月已支出金额，计算剩余预算
- 进度条分三级变色预警：
  - **绿色**（#00C897）：支出低于预算的 80%
  - **橙色**（#F59E0B）：支出在预算的 80%~100% 之间
  - **红色**（#FF6B6B）：支出超过预算 100%（超支状态）
- 账单变更后通过 `billsChanged` 信号自动刷新预算状态

### 2.6 多条件搜索筛选

- 支持四个维度的组合查询：
  - **关键字模糊搜索**：匹配备注字段
  - **收支类型筛选**：收入 / 支出 / 全部
  - **具体分类筛选**：下拉选择具体分类
  - **日期范围筛选**：自定义起止日期
- 搜索结果底部自动汇总：显示收入合计、支出合计及总结余
- 汇总行以灰色背景高亮，视觉上区分于数据行
- 重置按钮一键清空所有筛选条件

### 2.7 数据可视化

- **饼图（分类占比）**：按分类展示收入/支出金额占比，切片上显示分类名称和百分比，使用 15 色调色板确保各分类颜色区分，支持入场动画
- **折线图（日趋势）**：展示每日收支变化趋势，X 轴为日期（-45° 倾斜标签避免重叠），Y 轴自动适配数据范围，支持入场动画
- 支持切换收支类型和自定义统计时间范围，图表切换时自动刷新
- 基于 Qt Charts 模块实现，图表渲染使用反走样（Antialiasing）

### 2.8 全局主题与数据持久化

- **主题样式**：统一的现代简约平面设计风格，蓝色（#5B7FFF）为主色调，覆盖 QWidget、QTabWidget、QGroupBox、QPushButton、QLineEdit、QTableWidget、QProgressBar、QScrollBar、QComboBox、QDialog、QToolTip 等全部控件的样式
- **数据持久化**：基于 SQLite 本地数据库（`mymoney_data.db`），包含 5 张表（users、accounts、categories、bills、budgets），外键约束 + `ON DELETE CASCADE` 保证数据完整性

## 三、系统架构与模块设计

### 3.1 整体架构

本系统采用三层架构设计：

```
┌─────────────────────────────────────────────────┐
│                  表现层 (UI Layer)                │
│  LoginDialog / MainWindow / BillWidget / ...     │
│        CategoryManagerWidget / BudgetWidget      │
│        SearchFilterWidget / VisualizationWidget  │
│        AccountManagerWidget                      │
├─────────────────────────────────────────────────┤
│              数据库接口层 (Data Access Layer)      │
│              databaseManager (单例)               │
├─────────────────────────────────────────────────┤
│               存储层 (Storage Layer)              │
│               SQLite (mymoney_data.db)           │
└─────────────────────────────────────────────────┘
```

**设计要点**：

- **单例模式**：`databaseManager` 全局唯一，各 Widget 通过 `instance()` 静态方法获取引用，确保整个应用程序共享同一个数据库连接
- **信号槽解耦**：Widget 之间不直接通信，通过 Qt 信号槽机制经由 `MainWindow` 转发信号（如 `BillWidget::billsChanged` → `BudgetWidget::refresh()`），降低模块间耦合
- **三层分离**：表现层仅负责 UI 交互，数据库接口层封装全部 SQL 操作，存储层对上层完全透明

### 3.2 类协作关系图

```
main.cpp
  ├── LoginDialog ──→ databaseManager（登录验证 + 注册初始化）
  └── MainWindow
        ├── BillWidget ──→ databaseManager（账单CRUD + CSV导出）
        │     └── 信号 billsChanged() ──→ BudgetWidget.refresh()
        ├── CategoryManagerWidget ──→ databaseManager（分类CRUD + 图标管理）
        ├── BudgetWidget ──→ databaseManager（预算设置 + 统计查询）
        ├── AccountManagerWidget ──→ databaseManager（账户CRUD）
        ├── SearchFilterWidget ──→ databaseManager（多条件组合查询）
        └── VisualizationWidget ──→ databaseManager（分类/日统计数据）
```

### 3.3 核心类设计详述

#### 3.3.1 databaseManager（数据访问层单例）

| 属性 | 说明 |
|------|------|
| 继承关系 | 无（独立单例类） |
| 设计模式 | 单例模式（Meyer's Singleton） |
| 核心职责 | 数据库初始化、用户认证、CRUD 操作、统计查询、CSV 导出 |

**公开接口分类**：

- **用户管理**：`registerUser()`、`loginUser()`、`currentUserId()`、`currentUsername()`
- **账户管理**：`getAccounts()`、`addAccount()`、`updateAccount()`、`deleteAccount()`、`getOrCreateDefaultAccount()`
- **分类管理**：`getCategories()`、`getCategoriesByType()`、`addCategory()`、`updateCategory()`、`deleteCategory()`
- **账单管理**：`getBills()`、`getBillsByDateRange()`、`getBillsByType()`、`searchBills()`、`addBill()`、`updateBill()`、`deleteBill()`
- **预算管理**：`setBudget()`、`getBudget()`、`getMonthExpense()`、`getMonthIncome()`
- **统计分析**：`getCategoryStats()`、`getDailyStats()`
- **数据导出**：`exportToCSV()`

**关键实现细节**：

- 密码存储使用 `SHA-256(salt + password)` 哈希，盐值为编译期常量
- 账单增删改时通过事务性操作自动同步更新账户余额（`UPDATE accounts SET balance = balance ± amount`）
- 修改账单时先回滚旧账单的余额影响，再应用新账单的余额影响
- 分类查询使用 `COALESCE(icon, '')` 兼容旧数据库无图标字段的情况
- 数据库初始化时自动执行 schema 迁移（`ALTER TABLE ADD COLUMN icon`），保证向后兼容

## 四、关键技术实现

### 4.1 密码安全

密码采用 SHA-256 加盐哈希存储：`SHA-256("MyMoneyBook_Salt_" + password)`，盐值为编译期常量，不保存明文，防止彩虹表攻击。

### 4.2 账户余额自动同步

添加账单时根据收支类型自动更新账户余额（`balance ± amount`），修改账单时先回滚旧影响再应用新影响，删除时反向恢复。通过事务性 UPDATE 确保账户余额与账单数据始终一致。

### 4.3 动态 SQL 与参数化查询

搜索筛选功能根据用户输入动态拼接 WHERE 子句（关键字 LIKE、类型 =、分类 =、日期范围），全程使用 `QVariantList` 参数化查询，有效防止 SQL 注入。

### 4.4 数据库向后兼容

`init()` 中通过 `ALTER TABLE ADD COLUMN` 自动新增字段，并为旧数据的空图标字段批量补充默认值，保证已有用户数据库无缝升级，无需手动迁移。

### 4.5 全局主题样式

通过 `QApplication::setStyleSheet()` 在 `main.cpp` 中注入约 250 行 QSS 样式表，覆盖 QWidget、QTabWidget、QGroupBox、QPushButton、QLineEdit、QTableWidget、QProgressBar、QScrollBar 等全部控件。主色调 #5B7FFF，辅以 #F0F2F5 背景、#FFFFFF 卡片、#E4E7EC 边框，实现现代简约平面设计风格。

1. **Qt 信号槽机制**：深入理解了 MOC 元对象系统，掌握了组件间松耦合通信的设计方法。`billsChanged` 信号经 `MainWindow` 转发至 `BudgetWidget::refresh()`，各 Widget 无需相互持有引用。

4. **数据库向后兼容**：通过 `ALTER TABLE` 加条件 UPDATE 实现 schema 平滑升级，已有用户数据不丢失。
5. **Qt Charts 可视化**：掌握了 `QPieSeries`、`QLineSeries` 的图表渲染、动画、坐标轴定制等技能。

3. **导出格式**：目前仅支持 CSV，可扩展为 Excel 或 PDF 报表。
4. **测试覆盖**：缺少自动化测试，建议引入 Qt Test 框架对核心 CRUD 编写单元测试。
5. **图标扩展性**：emoji 方案简洁有效，后续可支持自定义图片上传以获得更专业的视觉效果。

## 附录：项目构建与运行

依赖 Qt 6.x（Widgets、Sql、Charts 模块）、CMake 3.16+ 及支持 C++17 的编译器。使用标准 CMake 流程构建（`cmake .. && cmake --build .`），首次运行自动创建 `mymoney_data.db` 数据库文件。
